UF2216. UD1. Actividad 1. Caso práctico: Creación de un repositorio para un sistema de información


Enunciado (Descripción)

En el siguiente supuesto práctico, realizar las funciones de creación de repositorio para cubrir las necesidades de gestión de información, de acuerdo al siguiente diseño:

Se quiere disponer de un mecanismo de almacenamiento de los elementos a gestionar en una empresa, con posibilidad de realizar cambios sobre los elementos almacenados y un registro histórico de las acciones realizadas con cada elemento o conjunto de elementos.

Los elementos a gestionar son archivos de texto, con distintos formatos.

También pueden ser documentos electrónicos o físicos, los documentos electrónicos contienen la URL de acceso al recurso junto con el usuario y password para acceder a su contenido.

Dichos documentos, tratan de manuales y documentación de programas y tecnología informática, es deseable que se separen por ámbitos.

Ver documento de apoyo: TortoiseSVN-1.8.11-es

· Si "x64 Edition" aparece en la lista debajo de Sistema ejecuta la versión de Windows XP de 64 bits.

Definición y explicación de la estructura

La estructura organizativa propuesta se divide en varios niveles y ámbitos para organizar y clasificar los documentos y archivos relacionados con la tecnología informática y los manuales de soluciones de software:

  • Proyecto 1: Este nivel representa el proyecto principal o la categoría general bajo la cual se agrupan todos los elementos relacionados.

  • Primer Nivel: Se trata de un subnivel que puede contener elementos más específicos o subproyectos relacionados con el proyecto principal.

  • Ámbito Tecnología Informática: Este ámbito se refiere a la categoría de documentos y archivos relacionados con la tecnología informática. Aquí se pueden almacenar manuales, documentación técnica, códigos fuente, scripts, entre otros recursos relacionados con la tecnología de la información.

  • Ámbito Manuales Soluciones Software: Este ámbito se refiere a la categoría de manuales y documentos relacionados específicamente con las soluciones de software. Aquí se pueden almacenar manuales de usuario, guías de instalación, manuales técnicos y cualquier otra documentación relacionada con las soluciones de software desarrolladas o utilizadas por la empresa.

Esta estructura organizativa proporciona una jerarquía clara y organizada para gestionar y acceder a los documentos y archivos relacionados con la tecnología informática y las soluciones de software en la empresa. Permite una fácil navegación y localización de los recursos relevantes, facilitando la gestión y colaboración entre los miembros del equipo.
